Java 는 bytecode 명령어로 이루어진 .class 까지 javac 로 컴파일링 한 뒤 JVM 위에서 한 줄씩 bytecode -> assembly code 로 interpreting 한다.

여기서 이 반복하는 곳을 런타임 때 어샘블리 코드로 미리 변환해두면 성능이 더 좋아질 것이라는 아이디어에서 JIT(Just-In-Time) 컴파일러가 등장했다. JIT 컴파일러는 자주 실행되는 바이트코드를 감지하여 네이티브 머신 코드로 변환하고, 이후 동일한 바이트코드가 실행될 때는 미리 변환된 네이티브 코드를 사용하여 성능을 향상시킨다.
